UT04 - Optimización y Documentación

Refactorización

La refactorización es un proceso esencial para mejorar la calidad del código, aumentar su legibilidad y mantenibilidad. Algunas características importantes son:

  • Se puede realizar durante o después del desarrollo del software.
  • Incluye técnicas como la reestructuración de código y la eliminación de duplicados.
  • Es importante realizar pruebas después de cualquier refactorización para asegurar que el comportamiento del software sigue siendo el esperado.

Patrones de Refactorización

Los patrones de refactorización son soluciones comunes y probadas a problemas específicos en el código. Algunos patrones comunes incluyen la extracción de método y la inversión de dependencias.

Utilizar patrones de refactorización es una buena práctica para mantener un código sólido y fácil de mantener.

Documentación

La documentación es esencial para un desarrollo de software óptimo. Es importante seguir estándares como Javadoc para generar documentación directamente del código en formato HTML.

Los tipos de comentarios Javadoc incluyen comentarios de línea y comentarios tipo C. Es fundamental conocer los tags de Javadoc para documentar adecuadamente el código.
